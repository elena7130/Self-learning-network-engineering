4. Increase the VLAN1 port priority of SW1's F0/1 to 240.Does SW3 select a different root port? Why/why not?

### Configuration

1. Identify the root bridge (Using CLI)

```
SW1 # show spanning-tree
SW1 # show spanning-tree detail
SW1 # show spanning-tree summary
```


2. on SW1 and SW2

```
SW1(config)spanning-tree mode pvst  
SW1(config)spanning-tree vlan 1 root primary
SW1(config)spanning-tree vlan 2 root secondary

SW2(config)spanning-tree mode pvst  
SW2(config)spanning-tree vlan 2 root primary
SW21(config)spanning-tree vlan 1 root secondary

```


 3. Increasing the cost of f0/2 on the SW4 to change the root interface 

```
SW4(config)interface f0/2
SW4(config-if)spanning-tree vlan 1 cost 100
```
*The original root interface in the VLAN 1 in the SW4 was f0/2 , but when I increase the cost of the f0/2 , the root interface in Vlan 1 become the f0/1 ,because the cost of f0/1 is low than f0/2's now . 

4. Increase priority of SW1's F0/1 to 240

```
SW1(config)interface f0/1
SW1(config-if)spanning-tree vlan 1 port-priority 240
```

5. Configure the portfast and BPDU guard on the interface f0/3 on the switch 

```
SW3(config)interface f0/3
SW3(config-if)spanning-tree portfast 
SW3(config-if)spanning-tree portfast bpdugard enable 
```
